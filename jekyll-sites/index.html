Jekyll is the most popular static site generator with over 30,000 stars on GitHub - for a reason: they offer free hosting for static sites.

When I first started putting my work to show, I had to cut costs and "settled" for a static site. Turns out, it's more of an alternative than a downgrade.

It provided me with a simple, file-based CMS that was easy to modify and update.

Jekyll uses the Liquid templating engine. It provides a lot of basic features for styling and modifying text. The more advanced features, such as includes and loops, turned out to be very powerful tools.

Not soon after my portfolio went public I was asked to do work on WeUseCoins (amongst others), one of the most popular sites about Bitcoin at the time.

A static website - generating a million views a month.

Our primary focus was on improving mobile interaction. The site was already fairly responsive, but infographics became unreadable at a small resolution and some elements just didn't scale correctly.

To ensure that everything was readable and looking nice, I converted some infographics to plain HTML & CSS and improved the outdated code to be more mobile-friendly.

As content was being added to the site, I created a wiki-like system entirely based on Jekyll. It allows easy adding and editing of persons and companies - but could be modified to include any other type of data.

With only one file per person, organization was as clean as it could have been without a real database.

I also created a JavaScript plugin for Jekyll to filter through a given dataset. On a smaller site, FarBeyondTheStars, I was able to integrate a whole-site search.

Still without a real backend - it was all done client-side. This approach would later contribute to my fascination of Vue.js and React.

Talk about limitations, huh?

During the development of a personal project I came up with a unique way to make Jekyll-generated sites multilingual. This translation system is still in use on WeUseCoins and BitcoinMining.

I was later invited to record a talk about it for JekyllConf but couldn't get it done in time (for which I am very sorry).
